Key Technical Decisions
Why Shopify Line Item Properties (not metafields)?

Metafields require additional API calls per order. Line item properties travel with the order natively, reducing API overhead and ensuring engraving data is never lost during order processing.

Why polling every 30s (not webhooks)?

Shopify webhook delivery is not guaranteed and requires HTTPS endpoint validation. Polling with intelligent caching gives 100% reliability with acceptable latency for an ops dashboard.

Why TOTP 2FA (not just JWT)?

The dashboard handles real customer order data. TOTP with backup codes meets the security standard expected for production admin tools managing PII and order financials.

  • Engineered a real-time engraving preview for 135+ cosmetic product SKUs — customers see exactly how their personalization looks before ordering. Per-product CSS positioning, 3 font families, 11 custom motifs via custom emoticon font, and shade-variant-aware rendering.

  • Architected a secure Node.js/Express REST API (20+ endpoints) with mandatory TOTP two-factor authentication (QR code setup, 8 backup codes, bcrypt-hashed), login rate limiting (10 attempts/15 min), role-based access control (Admin/User), and Helmet.js security headers.
